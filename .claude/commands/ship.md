現在の変更をコミット・プッシュして、GitHubにPRを作成してください。

## 手順

1. `git status` と `git diff` で変更内容を確認する
   - 変更がなければ「コミットする変更がありません」と伝えて終了する
2. `git log --oneline -5` で最近のコミットスタイルを確認する
3. 変更内容を分析して日本語のコミットメッセージを生成する
   - 形式: `<種別>: <何を・なぜ>`（例: `UI: 音声信頼度の閾値表示を修正`）
   - 種別: 機能追加=機能名、バグ修正=`fix`、リファクタ=`refactor`、UI変更=`UI`
4. 生成したコミットメッセージをユーザーに提示して確認を求める
5. 承認されたら以下を実行する：
   - `git add` で関連ファイルを追加（`.env` 等の機密ファイルは除外）
   - `git commit -m "<メッセージ>\n\nCo-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>"`
   - `git push -u origin <branch>`
6. `gh pr create` でPRを作成する
   - タイトル: コミットメッセージの1行目
   - 本文: Summary（変更点3点以内）+ Test plan（確認項目チェックリスト）
7. PR URLをユーザーに伝える
